# Soundwich 🥪

**맥(macOS)을 위한 앱별 오디오 라우팅 도구.** 스포티파이는 블루투스 스피커로,
화상회의는 에어팟으로, 나머지는 맥북 스피커로 — 동시에 각자 다른 기기로 소리를
내보낼 수 있어요.

> _소리의 샌드위치 — 각 앱의 소리를 제자리로._

[English](README.md) · 한국어

![앱 3개를 서로 다른 출력 장치로 라우팅한 Soundwich 화면](docs/screenshot.png)

## 무엇을 하는 앱인가요

맥은 기본적으로 **전체 시스템에 출력 장치 하나**만 고를 수 있어요. Soundwich는 그
한계를 넘습니다 — 애플의 최신 **CoreAudio Process Tap API**(macOS 14.2+)를 이용해
**앱마다 원하는 출력 장치로** 소리를 보내요. 가상 오디오 드라이버 설치도 필요 없습니다.

- 🎯 **앱별 라우팅** — 앱마다 출력 장치를 따로 지정
- 🔀 **동시 라우팅** — 스포티파이 → 스피커, 크롬 → 헤드폰, 동시에
- 🖥️ **시스템 출력 전환** — 설정 앱 안 열고 기본 장치 변경
- 💾 **설정 기억** — 앱별로 저장, 다시 켜도 유지
- ✋ **완전 수동** — 절대 알아서 바꾸지 않음, 항상 사용자가 제어
- 🎧 **기기 교체 대응** — 에어팟을 빼면 소리가 자연스럽게 폴백되고, 다시 끼면 라우팅
  자동 복원
- 🚀 **로그인 시 시작** — 선택 사항, 한 번 켜두면 끝

## 설치

1. [Releases](https://github.com/zzanggi-haki/soundwich/releases)에서 최신
   **`Soundwich.dmg`**를 받으세요.
2. 열어서 **Soundwich**를 **응용 프로그램**으로 드래그.
3. 무료 미서명 빌드라 macOS Gatekeeper를 처음 한 번 우회해야 해요 —
   **[INSTALL.md](INSTALL.md)** 참고 (터미널 한 줄이면 끝).
4. 처음 사용할 때 **시스템 오디오 녹음** 권한을 허용하세요. 앱 소리를 캡처해서
   다른 기기로 보내기 위한 권한입니다.

**요구 사항:** macOS 14.2 이상 · Apple Silicon 또는 Intel (universal 빌드).

## 사용 방법

1. 메뉴바의 🥪 아이콘을 클릭.
2. **Apps** 섹션에서 소리를 옮기고 싶은 앱을 찾으세요 (소리를 내기 시작하면 목록에
   나타나요).
3. 그 앱의 장치 드롭다운을 눌러 어디로 보낼지 선택.
4. 끝 — 그 앱은 이제 선택한 기기로 소리가 나와요. 원하는 만큼 여러 앱에 반복하세요.

라우팅을 끄려면 드롭다운에서 **제어 해제**를 선택하세요. 맨 위 **System** 섹션에서는
나머지 모든 소리의 기본 출력 장치를 바꿀 수 있어요.

## 소스에서 빌드

```sh
brew install xcodegen        # 최초 1회
xcodegen generate            # Soundwich.xcodeproj 생성
open Soundwich.xcodeproj      # 이후 ⌘R
```

배포용 universal `.dmg` 만들기:

```sh
./scripts/build_dmg.sh       # → Soundwich.dmg (arm64 + x86_64)
```

## 동작 원리

대상 앱에 **Process Tap**을 걸고, 그 탭을 선택한 출력 장치로 향하는 비공개
**aggregate 오디오 장치**로 감싼 뒤, 실시간 IO 콜백에서 캡처된 오디오를 그대로
흘려보내요. 라우팅은 앱 번들 ID 기준이며 그 앱의 모든 프로세스를 한 번에 탭합니다 —
브라우저 소리(헬퍼 프로세스가 재생)가 안정적으로 잡히는 이유예요.
